# Назначение ФОС

Фонд оценочных средств предназначен для оценки результатов освоения дисциплины «Методы оптимизации в системах искусственного интеллекта» при выполнении проектной работы.

Основное внимание уделяется не только корректности реализации алгоритмов оптимизации, но и способности студента аргументированно выбирать методы, анализировать результаты экспериментов, обеспечивать воспроизводимость исследования и учитывать вопросы устойчивости моделей машинного обучения.

ФОС разработан по принципу «зеркальной» структуры репозитория, благодаря чему преподаватель и студент работают с одинаковой архитектурой проекта и единым набором инженерных артефактов.

---

# Профильные роли КРМ

В рамках дисциплины оцениваются компетенции по двум профессиональным ролям.

## ML Researcher

Роль ориентирована на разработку, исследование и сравнение алгоритмов оптимизации для задач машинного обучения.

В ходе выполнения проекта студент должен уметь:

- выбирать алгоритмы оптимизации под поставленную задачу;
- реализовывать их программно;
- анализировать скорость сходимости и качество решения;
- проводить сравнение с базовыми подходами;
- делать выводы по результатам вычислительных экспериментов.

## AI Security Engineer

Роль связана с обеспечением устойчивости ML-моделей в процессе обучения.

Оценивается способность:

- анализировать устойчивость алгоритмов;
- учитывать влияние шума и нестабильности данных;
- применять методы робастной оптимизации;
- выявлять потенциальные риски деградации модели;
- обосновывать выбранные меры повышения надежности.

# Оценочные средства

Для оценки результатов обучения используются несколько взаимодополняющих инструментов.

| Код | Оценочное средство | Назначение |
| --- | --- | --- |
| ОС-1 | Лабораторные работы | Проверка владения отдельными методами оптимизации и практических навыков программирования |
| ОС-2 | Курсовой проект | Комплексная оценка реализации алгоритмов, качества исследования и документации |
| ОС-3 | Peer-review | Оценка воспроизводимости проекта, качества документации и инженерной культуры |
| ОС-4 | Экзамен | Проверка понимания теоретических основ и способности анализировать методы оптимизации |

Каждая ключевая компетенция дисциплины подтверждается минимум двумя различными оценочными средствами.

# Оси оценивания курсового проекта

Проект оценивается независимо по двум основным осям.

## 1. Программное решение (MVP)

Оцениваются:

- корректность реализации алгоритмов;
- качество программного кода;
- достижение целевых метрик;
- эффективность выбранного метода оптимизации;
- корректность постановки и проведения вычислительных экспериментов;
- устойчивость решения к изменению входных данных.

## 2. Документация

Оцениваются:

- полнота описания проекта;
- обоснование выбора методов;
- анализ полученных результатов;
- качество оформления репозитория;
- воспроизводимость экспериментов;
- описание ограничений и направлений дальнейшего развития.

# Шкала оценивания

| 10-балльная шкала | 5-балльная система | Интерпретация |
| ---: | :---: | --- |
| 10 | 5 | Отличный результат, требования выполнены полностью |
| 8–9 | 5 | Высокий уровень, имеются несущественные замечания |
| 6–7 | 4 | Хороший результат, отдельные элементы требуют доработки |
| 4–5 | 3 | Минимально достаточный уровень освоения дисциплины |
| 0–3 | 2 | Компетенции не сформированы в требуемом объеме |

---

# Порог успешного освоения дисциплины

Дисциплина считается освоенной при одновременном выполнении двух условий:

- итоговая оценка составляет не менее **4,0 баллов**;
- по каждой из двух осей («Программное решение» и «Документация») получено **не менее 4 баллов**.

Получение высокого среднего балла не компенсирует неудовлетворительный результат хотя бы по одной из осей.
